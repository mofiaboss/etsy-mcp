# apps/etsy

The etsy-mcp MCP server. Wraps Etsy Open API v3 as 90 pure-primitive tools across 13 categories. Built on top of the workspace packages `etsy-core` (HTTP, OAuth, retry, rate limit, redaction) and `etsy-mcp-shared` (permissions, confirmation, lazy loading).

## Layout

```
apps/etsy/
  src/etsy_mcp/
    __main__.py           # CLI entry point
    main.py               # FastMCP server bootstrap
    runtime.py            # @lru_cache singleton factories
    bootstrap.py          # config/auth/client wiring
    categories.py         # ETSY_CATEGORY_MAP + TOOL_MODULE_MAP
    schemas.py            # JSON schemas for tool input validation
    cli/                  # auth login, status, logout
    config/               # default config.yaml
    managers/             # one per domain (shop, listing, receipt, ...)
    tools/                # thin tool wrappers (one file per category)
    models/               # shared pydantic models for field symmetry
    tools_manifest.json   # generated by `make manifest`
  tests/
    unit/                 # fast, no network
    integration/          # gated on ETSY_INTEGRATION_TESTS=1
  Makefile                # app-level commands
  pyproject.toml
```

## Run

```bash
# Authenticate (one-time)
etsy-mcp auth login

# Run the server (stdio mode)
uvx etsy-mcp@latest

# Or against the workspace
uv run --package etsy-mcp etsy-mcp
```

## Configuration

| Variable | Required | Description |
|---|---|---|
| `ETSY_KEYSTRING` | Yes | Etsy app keystring (client ID) from https://www.etsy.com/developers |
| `ETSY_REFRESH_TOKEN` | No | Headless mode — bypass `auth login` by providing a pre-obtained refresh token |
| `ETSY_TOKEN_STORE` | No | Override the tokens.json path. Default: `~/.config/etsy-mcp/tokens.json` |
| `ETSY_TOOL_PERMISSION_MODE` | No | `confirm` (default) or `bypass` |
| `ETSY_POLICY_<CATEGORY>_<ACTION>` | No | Hard policy gates (e.g., `ETSY_POLICY_LISTINGS_DELETE=false`) |
| `ETSY_DAILY_BUDGET` | No | Override daily request budget. Default: 10000 |
| `XDG_CONFIG_HOME` | No | Standard XDG config root. Default: `~/.config` |
